## ðŸ› ï¸ n8n Workflow Architecture

The project is structured as a multi-agent ecosystem interacting through specialized n8n nodes:

### 1. Gatekeeping & Strategy
* **Strategist (Mistral Agent):** Evaluates if the user's claim is specific enough for a forensic audit.
* **Logic Gate (If Node):** If the claim is vague, it triggers a `CLARIFY` loop; if ready, it proceeds to the research stage.

### 2. Forensic Research Engine
* **HTTP Request:** Executes a live search on the ArXiv API using optimized keywords generated by the Strategist.
* **XML-to-JSON Parser:** Converts raw scientific XML data into processable JSON objects for the AI agents.
* **Batch Looping:** Processes each research paper individually using a `Split In Batches` node to ensure deep analysis of every source.

### 3. The Methodology Auditor (The "Hard" Part)
This is the core technical innovation where a **Custom Code Node (JavaScript)** executes a mathematical audit on the findings:
* **Recency Boost:** Studies from 2025/2026 receive a +30 point boost to prioritize cutting-edge data.
* **Peer-Review Validation:** Checks the `arxiv:journal_ref` parameter to confirm publication in recognized scientific journals (+20 points).
* **Red Flag Detection:** Automatically penalizes studies for small sample sizes or if testing was limited to "mice/in-vitro" (-20 points).

### 4. Intelligence Briefing
* **Final Auditor Agent:** Synthesizes all audited data into a definitive briefing without "sugar-coating," utilizing a formal **Evidence Pyramid** (ranging from Theoretical to Empirical and Peer-Reviewed).

## Credibility algorithm

- **Source weight:**  
  `credibilityWeight = (log(citationCount + 1) * studyTypeWeight) / (currentYear - year + 1)`

- **Study type weights:** Meta-analysis 5, RCT 4, Cohort 3, Observational 2, Opinion 1.

- **Consensus score:** Weighted average of synthetic alignment (0â€“100%) across sources.

- **Misinformation risk:** Inferred from consensus score, number of sources, and share of opinion-type sources (Low / Medium / High).

## API

- **POST /webhook/truthlens** (n8n â€“ used by frontend)  
  Body: `{ "message": "Your claim.", "id": "uuid", "timestamp": "2025-02-28T12:00:00.000Z" }`  
  Returns: `{ "analysis": AnalysisResult }` or a raw `AnalysisResult` (from n8n Respond to Webhook).

- **POST /api/claims/analyze** (backend)  
  Body: `{ "text": "Your scientific claim here." }`  
  Returns: `{ "analysis": AnalysisResult }` (claim, risk, score, sources, distributions, knowledge gaps, fact/opinion breakdown).

- **GET /health**  
  Returns `{ "status": "ok", "service": "truthlens-api" }`.
